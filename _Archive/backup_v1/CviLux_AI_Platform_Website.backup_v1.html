STRATEGIC POSITIONING · 策略定位

六大設計總原則

v2.0 強化 EIP 對接與 Multi-Agent 治理，但仍堅守「不重做既有系統、AI 輔助而非自動決策」的核心原則。

1. 不重做既有系統

ERP、D365、M365、MES、EAP、EIP 皆保留原主責；本平台僅為前台 + AI Agent + 整合中介。

2. 網站作為入口

對外行銷 / Lead / 產品搜尋；對內 AI 查詢 / 審核 / 預警；EIP 作為單一身份與工作流主軸。

3. API-First

所有跨系統交換經 API Gateway / EIP 連接器；不直連資料庫，避免緊耦合。

4. AI 輔助，不自動決策

產品建議、報價、ESG 文案皆 AI 草稿；商業重要決策（成交、發布、簽約）必經人工審核。

5. 內外網分層

外網對客戶；內網對員工 + 審核 + 治理；EIP 作為內網單一登入與工作流核心。

6. 資料可追溯

所有文件、報價、AI 回答、Agent 決策皆保留 Log + Audit Trail，符合 ISO 27001 與內稽要求。

EIP INTEGRATION HUB · v2.0 NEW

EIP 整合中介層 — 內網共用作業中樞

EIP（Enterprise Information Portal）作為瀚荃集團「共用作業系統」核心，統一員工身份、組織、簽核流程、行事曆、訊息、文件入口，並以單一介面串接 ERP、D365、M365、EAP、Azure Fabric。本平台對內入口（第 2 層）以 EIP 為登入主軸；AI Agent 工作台、報價中心、文件審核中心皆內嵌於 EIP 應用區。

EIP 在平台的 6 大角色
  • SSO 單一登入 — Azure AD / Entra ID 主控
  • 組織同步 — BU、部門、角色、主管樹
  • 簽核流程 — 文件 / 報價 / 促銷 / ESG
  • 行事曆與會議 — Outlook / Teams 整合
  • 訊息中心 — Teams / WeCom 跨域推播
  • 應用入口 — 各內網應用統一入口

TASK ROUTING FLOW

編排者任務派工流程

  1. 1
    意圖辨識 Intent Detection
    Orchestrator 解析使用者輸入（自然語言、圖片、表單）；判定屬於技術 / 銷售 / ESG / 行銷 / 報價 / 維運 等任務類別。
  2. 2
    任務拆解 Decomposition
    複雜任務拆為子步驟（如：客戶上傳圖片 + 詢價 + 替代料 → 圖搜圖 + 報價 + Cross-ref 三 Agent 平行執行）。
  3. 3
    並行 / 串行派工
    獨立任務並行；有依賴關係（如報價需要先取得替代料規格）改串行。
  4. 4
    Human-in-the-loop 關卡
    觸發以下情境必停下：超毛利底線報價、對外發布內容、修改 ERP 資料、跨境資料傳輸、刪除動作。
  5. 5
    結果整合 + Audit Log
    Orchestrator 彙整 Agent 回應 → 統一格式 → 寫入 Audit Log（含 prompt、source、決策、最終回覆、簽核者）。
GOVERNANCE & GUARDRAILS

Agent 治理與護欄

Hallucination 防範
所有事實性回答必引用 SharePoint / ERP / D365 來源 ID；無來源 → 回覆「無法驗證」。
權限分級執行
Agent 透過呼叫者的 Azure AD token 取得資料；不得越權；NDA 文件需二次驗證。
禁止動作清單
永遠禁止：刪除 ERP/CRM 資料、自動發送外部 eDM、修改員工權限、修改安全規則、跨境輸出敏感資料。
可觀測性 Observability
每次 Agent 呼叫記錄 latency、token、cost、信心分數、引用源；異常自動通報 Ops Agent。
持續優化迴圈
使用者 thumbs up/down → 建立 RLHF 樣本 → 月度評估 Agent 準確率與改善 prompt。

推論 Multi-Agent 協同設計引用 LangGraph / OpenAI Agents SDK / Microsoft Copilot Studio Multi-Agent / Google Vertex AI Agent Builder 之公開觀察模式。實作技術選型須由 IT 評估後決定。

EXECUTIVE SUMMARY · ENGLISH

CviLux Group AI Digital Operation & Global Marketing Platform v2.0

PLATFORM POSITIONING

A unified AI front-end marketing portal + internal collaboration hub + integration middleware for the CviLux Group ecosystem (CviLux components, CviCloud sCMS, EAP ESG SaaS, Opro9 consumer, CviMall commerce). The platform does not replace existing ERP, D365 CRM, M365 SharePoint, MES, or EAP systems — it connects them through APIs and AI agents, eliminating duplicate maintenance.

v2.0 KEY ENHANCEMENTS
  • EIP Integration Hub — promoted to a standalone layer, with vendor selection matrix (M365 + Power Platform vs. DingNew EIP vs. Custom-build vs. Third-party SaaS).
  • Multi-Agent Orchestration — 1 Orchestrator + 9 specialist agents (Tech, Sales, ESG, Image-Search, Cross-ref, Quote + new Marketing, Brand-Voice, Ops).
  • Brand Auto-Marketing Engine — 7-step pipeline from topic ingestion → multilingual content → brand-voice review → human approval → multi-channel publish → lead nurture & scoring.
  • Governance & Guardrails — RAG-only factual answers, RBAC enforcement, hard-blocked actions, observability, RLHF feedback loop.
DESIGN PRINCIPLES

(1) Do not rebuild existing systems. (2) Portal-first approach — external for marketing, internal for collaboration. (3) API-first integration. (4) AI as advisor, not autonomous decision-maker. (5) Strict separation of external & internal layers. (6) Full audit traceability for documents, quotes, AI answers, and agent decisions.

ROADMAP

Phase 1 (0-9m) — multi-brand site, CviLux search, SharePoint integration, D365 Lead, ERP inventory, AI agents MVP, EIP SSO. Phase 2 (10-18m) — CviMall transactions, eDM automation, training academy, sCMS/EAP funnels, dealer inventory sharing, Multi-Agent Orchestrator. Phase 3 (19-24m) — China market via WeCom, dealer collaboration, cross-border compliance (PIPL), regional EIP deployment.

CORE VALUE
  • Convert product data into searchable, quotable, transactable marketing assets.
  • Translate ESG and CBAM capabilities into competitive export advantages.
  • Deploy AI for sourcing, document retrieval, quoting, and lead management.
  • Monetize ERP slow-moving inventory through dealer and channel offers.
  • Convert internal documents and tacit knowledge into auditable, traceable digital assets.

Inferred Vendor evaluation, timeline estimates, and architectural patterns are recommendations based on the attached design document and observed industry practice. Actual implementation requires IT & BU RFP evaluation and POC validation.
