一、結論摘要（Executive Summary）

P01「AI 數位營運與全球行銷平台」是集團對外的「入口層」—— 把 6 大品牌、既有 ERP/D365/M365/EAP 整合於一個前台，並以 9+1 Multi-Agent 串接內外網作業。 EAP 則是CviCloud 對客戶銷售的「產品層」—— 一條完整的 SaaS + IoT 硬體產品線，由 5 個系統模組（EMS、MES、EHS、CMS、SCADA）+ 13 個 IoT 硬體 SKU 組成。

P01 在 EAP 觀點的角色

  • P01 是 EAP 的銷售前台與品牌入口
  • P01 的 Lead 系統 = EAP SaaS 試用申請通道
  • P01 的 ESG Agent 應呼叫 EAP CMS 推論 API

EAP 在 P01 觀點的角色

  • EAP 是 P01「品牌矩陣」中的一張卡（Layer 6.7）
  • EAP 細節（5 模組 + 13 SKU + 10 套裝）目前在 P01 中尚未充分呈現
  • EAP 的推論 API 應成為 P01 Multi-Agent Orchestration 的可呼叫工具

推論 核心發現：P01 與 EAP 並非競合，而是「外殼 vs 內核」關係。 P01 是集團統一品牌與 AI 治理層；EAP 是 CviCloud 銷售給客戶的 SaaS+硬體商品。 兩者整合好則 P01 成為 EAP 全球行銷與銷售引擎，EAP 成為 P01 ESG/IoT/碳管能力底座； 整合不好則出現「P01 模糊提及 EAP」「EAP 自己做小網站重複行銷」之雙頭情況。

八、雙方缺口

8.1　P01 中「EAP 描述不足」之處（待強化）

  1. EAP 卡僅 5 行字：P01 的「品牌矩陣 EAP 卡」目前只描述「多租戶 SaaS · sEMS / sCMS / sMES / sEHS / sWMS · 模組訂閱」—— 未驗證 但實際 EAP 是「EMS / MES / EHS / CMS / SCADA」；P01 所列模組名稱與 EAP 規範不一致。建議更正。
  2. 無 EAP 之 IoT 硬體呈現：P01 沒有展示 EAP 13 個 SKU；對外網站缺少完整產品圖庫。
  3. 無銷售套裝路徑：P01 未呈現 EAP 之 10 種套裝組合，無法引導客戶選型。
  4. Multi-Agent 未呼叫 EAP API：9+1 Agent 中 ESG Agent 只提及 sCMS / EAP CBAM，未說明如何透過推論端點 API 取得實際碳排數值。
  5. 無 EAP 試用 / 訂閱流程設計：P01 提到「EAP 導流、SaaS 試算、試用申請」但缺少具體流程與 D365 對接欄位。
  6. 無 EAP × 鼎新永續數據館 接點：P01 僅提鼎新 ERP 與鼎新 MES，未涵蓋鼎新永續數據館（EAP 已規劃對接）。
  7. 無 LifeSmart 整合提及：EAP 規劃串接 LifeSmart 完成智慧連動；P01 未提。

8.2　EAP 中「P01 角色缺失」之處（待強化）

  1. 無對外行銷入口設計：EAP 規劃無「客戶從哪認識 EAP」之路徑；應指向 P01 的 EAP Landing。
  2. 無 EIP 整合：客戶採購 EAP 後，集團內部使用是否走 EIP SSO？尚未對齊。
  3. 無 Multi-Agent 視覺：EAP Dashboard 自然語言查詢 ≠ Multi-Agent Orchestration；可考慮把 EAP 的對話層接到 P01 的 Orchestrator。
  4. 銷售路徑與 D365 整合不明：EAP 試用 → 簽約 → 部署 → 續約之 Lead → Opportunity → Quote 流程，與 P01 的 Lead 系統如何串？
  5. 客戶教育訓練：EAP 客戶教育是否走 P01「瀚荃學苑外網」？或自建？
  6. P01 行銷流水線可賦能 EAP 內容生成：EAP 客戶案例、白皮書、CBAM 解決方案 eDM，可由 P01 行銷自動化 Agent 量產。
