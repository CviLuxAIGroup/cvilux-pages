共同經營理念

ESG · 碳淨零 · 共同經營理念

「Connect the World · Conserve the Earth」不只是標語 — 是**兩個網站的共同骨架**，從導覽結構、視覺、內容、KPI 全面貫穿。

主題 1：產銷淨零循環

CviLux **製造端**自身淨零實踐 → CviCloud **解決方案** → 賦能客戶淨零 → 客戶下單 CviLux 連接器 → 形成閉環。

兩站共同講「我們做到了，你也可以」的故事。

主題 2：21 年 ESG 信用

2003 起 ISO 14001、14064、14067、50001 等認證歷程是 CviCloud 對客戶的最大說服力。

CviLux 證明「能做到」→ CviCloud 提供「做到的工具」

主題 3：CBAM 一站服務

2026 起 CBAM 全面開徵，**所有出口歐盟客戶**都需要碳足跡報告 — CviLux 自己會做、CviCloud 賣這個能力。

外銷集團需求 + ESG SaaS 需求一站滿足

SHARED ESG HUB · 共用 ESG 中心

建議設計：esg.cvilux-group.com 為集團 ESG 中央資源

兩站均連結到同一個 ESG Hub，避免重複建設且統一品牌訊息。內含：

📊 集團 ESG 報告書（年度）
📜 ISO 14001 / 14064 / 14067 / 50001 認證
📑 CBAM 申報範本與指南
🌱 集團淨零路徑圖（SBTi 已申請否?）
🏆 客戶 ESG 評鑑成績（EcoVadis / CDP）
📚 ESG / CBAM 知識文章
📺 永續長媒體訪談影片
🤝 客戶減碳成功案例

sCMS 商模 4 方案

CviCloud sCMS Business Model

CviCloud sCMS 商模仍在考慮中。下列 4 方案各有優劣 — 每個方案對網站設計、客戶旅程、收入結構、合作對象（騰雲）都有不同要求。

OPTION A

純硬體賣斷

Hardware-Only

內容

客戶買感測器、閘道、Edge 設備；自己接雲、自己分析

收入一次性
毛利低-中（30-40%）
維運客戶自管
客戶金流Capex
複雜度低
適合：小型 / 不熟雲端客戶
OPTION B

SaaS 軟體服務

Software-as-a-Service

內容

客戶用自家 IoT 或第三方硬體；訂閱 sCMS 雲平台分析 / 報表

收入**經常 ARR**
毛利高（70-85%）
維運瀚雲（or 騰雲）
客戶金流Opex
複雜度中（需經營雲）
適合：中大型已有 IoT 客戶
OPTION C

HaaS 硬體即服務

Hardware-as-a-Service

內容

客戶租賃硬體 + 軟體訂閱包月；含安裝、維護、升級全包

收入**經常 + 高黏著**
毛利中（40-55%）
維運瀚雲全包
客戶金流Opex
複雜度**高**（融資 + 物流）
適合：中小型客戶（無 IT 部門）
OPTION D ⭐ 建議

混合分層

Hybrid Tiered

內容

大客戶：賣斷+SaaS / 中客戶：純 SaaS / 小客戶：HaaS — 一站滿足

收入三組合並行
毛利加權 50-65%
維運分工模式
客戶金流客戶選擇
複雜度**最高**（管理 3 模式）
⭐ 配合 ESG 普及最廣
IMPACT ON WEBSITE · 各商模對網站設計的影響
設計項目 A 賣斷 B SaaS C HaaS D 混合 ⭐
產品頁產品型錄SaaS 模組介紹套餐頁客戶分層導向
CTA詢價註冊試用租賃申請**3 個 CTA 並列**
會員系統不需**SSO + Plan**合約管理混合會員
客戶後台無**自助管理**合約 + 工單三合一
金流整合B2B 對帳**訂閱+信用卡**分期租賃3 種金流
建置複雜度低中高最高
建議網站階段P1 立即可上P2 後續P3 長期**P1 視覺先 / P2-3 漸進**
⭐ 我的建議：D 混合分層 + 漸進建置

網站設計時即預留 3 種商模，但第一期只先上 A 賣斷 + B SaaS Demo 申請。HaaS 列為第二期。原因：

  • 客戶分層自然存在，不能逼客戶接受同一商模
  • HaaS 涉及融資、物流、合約複雜，第一期就上會拖慢主流程
  • SaaS 為高毛利、可規模化、ARR 穩定收入，應該優先佈局
  • 賣斷立即可變現、客戶最熟悉的模式
  • 「ESG 即服務（Sustainability-as-a-Service）」可成為對外行銷主標語

5 大融合協作點

Fusion Touchpoints

兩個獨立網站如何融合協作？以下 5 個觸點是設計重點 — 既保留各自獨立、又讓客戶旅程順暢。

FUSION 1

共用 ESG Hub

建立 esg.cvilux-group.com 集中所有 ESG 內容；兩站都嵌入此 Hub 而非各自做。

效益：避免重複建設、統一訊息、SEO 集中

FUSION 2

共用導覽列

兩站頂部 nav 永遠看得到對方品牌的入口（如 CviLux 站永遠有 sCMS 連結）。

效益：客戶任何時候都知道集團還有什麼

FUSION 3

共用 SSO + 客戶資料

客戶在 CviLux 註冊（RFQ）→ 同帳號可登入 CviCloud 試用 sCMS；資料統一進 D365。

效益：降低重複註冊摩擦、客戶資料完整

FUSION 4

交叉 Lead 導流

CviLux 客戶有 ESG 需求 → 自動推薦 CviCloud；CviCloud 客戶採購零件 → 推薦 CviLux。

效益：擴大集團錢包份額（Wallet Share）

FUSION 5

共用 AI Agent

同一個 ESG Agent 服務兩個網站；技術 Agent 同時處理連接器與 sCMS 問題。

效益：知識共享、開發成本減半

不融合的部分

獨立元素

產品搜尋（兩站完全不同）、後台管理、各自客戶後台、各自定價系統 — 這些不需要融合。

原則：使用者體驗融合，但系統耦合最小化

05

客戶旅程整合

Customer Journey Integration

以「歐盟出口客戶 + CBAM 申報需求」為案例，展示客戶在兩個網站的順暢旅程。

CASE: EUROPE OEM CUSTOMER · 案例：出口歐盟之代工廠採購主管
1
CviLux 站客戶來源：Google 搜尋連接器規格

客戶搜尋 Type-C 連接器規格 → 進到 CviLux 產品頁 → 用 AI 圖搜圖找到符合產品 → 下載 Spec → 送出 RFQ

2
CviLux 站RFQ 表單

表單中**新增一題**：「您的最終產品是否出口歐盟？需要 CBAM 碳足跡資料嗎？」 → 勾選後系統紀錄

融合協作點⭐ 跨站協同自動化

銷售 Agent 自動執行：發送個人化 eDM 給客戶，附 (1) CviLux 報價（含產品 carbon footprint）+ (2) CviCloud sCMS 介紹「我們客戶用這套工具 12 個月內取得 CBAM 申報能力」

4
CviCloud 站客戶點 eDM 連結進入

客戶到 sCMS 頁面 → 看 CBAM 解決方案 → 用 ROI 試算機輸入廠別、產量 → 看到「3 個月回本」估算 → 申請 Demo

5
CviCloud 站Demo 申請

系統認出客戶（透過 SSO 從 CviLux 帶入）→ 直接用既有資料，不用重填 → 客戶 Demo 後選 SaaS B 方案

結果⭐ 集團錢包份額擴大

**單一客戶**從 CviLux 進入，最終購買：(1) 連接器年度合約 NT$ 500 萬 + (2) sCMS SaaS 訂閱 NT$ 50 萬/年 = **集團合計 ~550 萬**，且形成長期合作關係

時程與里程

Timeline & Milestones

兩網站不是同時上線，而是分期建置：先 CviLux 主站打地基，再上 CviCloud（避免一次太多風險）。

PHASE 1 · M1-M5（5 個月）

CviLux 綜合網站 MVP

  • · 集團首頁 + 控股結構
  • · CviLux 產品搜尋 + AI 圖搜
  • · RFQ → D365
  • · SharePoint 文件下載
  • · ESG Hub 雛形
  • · **CviCloud 暫只放介紹頁 + Demo 申請表單**
PHASE 2 · M6-M9（4 個月）

ESG Hub 建構 + 交叉導流

  • · 完整 ESG Hub 上線
  • · 兩站交叉導覽連結
  • · 銷售 Agent 跨站推薦
  • · CviCloud sCMS 主站開始建置（A 賣斷）
  • · 客戶案例 / Reference 上架
PHASE 3 · M10-M14（5 個月）

CviCloud sCMS 完整上線

  • · SaaS 模式上線（會員 + 訂閱 + 後台）
  • · SSO + 跨站客戶資料同步
