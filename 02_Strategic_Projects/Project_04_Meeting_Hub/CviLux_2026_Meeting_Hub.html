08

Q&A 預備

Anticipated Questions

主持人事先準備 10 個高機率問題與簡短回答，會議現場可快速回應，避免冷場或失焦。

Q1：為什麼是 1,010-1,440 萬？這個範圍怎麼算？
A：PoC#1 約 430-590 萬（資料工程 + ML + 顧問 + 內部資源）；PoC#2 約 580-850 萬（含 IoT 部署 100-180 萬）。範圍取決於最終廠商報價、IoT 點位數、是否啟用視覺品檢。詳見 PoC 評估書 §5。
Q2：為什麼選東莞而不是淡水做試點？
A：東莞為集團主製造廠（量大、料號多、資料齊），跑出 PoC 結果可推 2-3 廠；淡水雖在台灣便於配合，但量體較小不利於模型訓練。如有 PIPL 顧慮，先做台灣端（淡水）也可接受，但需重新評估資料量是否足夠。
Q3：WF AIGP 還在升級中，會不會影響進度？
A：本案 Phase 1 不依賴 WF AIGP，先用鼎新 ERP 既有 API + 中介整合層；WF AIGP 上線後再透過 Webhook / API 漸進整合。即使 WF AIGP 延誤 6 個月也不影響 Phase 1 上線。
Q4：AllSor 為什麼從關係企業改成非集團？
A：依董事長 2026-05-03 指示，AllSor 瀚碩科技股份有限公司在法律上獨立於瀚荃集團，未涉股權關係。雙方僅為協同行銷夥伴。CIS v1.2 已正式更正，所有對外文宣須清楚標示「非集團公司」。
Q5：AI 會不會自動修改 ERP 資料？
A：絕對不會。所有 AI 建議必經 Human-in-the-loop 人工審核才執行；超毛利底線報價、修改 ERP 主檔、發布外部訊息、刪除動作都是「永遠禁止 Agent 自動執行」之清單。詳見 AI 平台網站 §6 Multi-Agent Guardrails。
Q6：6 個月做 Phase 1 MVP 太樂觀？
A：MVP 範圍刻意縮小 — 只做：多品牌首頁 + CviLux 產品搜尋 + RFQ + D365 寫入 + ERP 庫存讀取 + AI 基礎 RAG。不做 CviMall 完整交易、企業微信、AI 全自動報價。如資料盤點順利，6 個月可達；如資料品質差，先做 2-4 週「資料修補 Sprint」。
Q7：如果 PoC 失敗怎麼辦？
A：PoC 結束時技術 KPI 全部達標 + 商業 KPI 至少 2 項達標 + 使用者滿意度 ≥ 3.5 才進量產化；否則啟動「為何失敗」回顧 + 決定終止 / 延長 / 重設範圍。即使失敗，集團累積「製造業 AI 落地能力」與資料治理基礎仍是正資產。
Q8：陸區資料能不能傳台灣？
A：原則上陸區資料留陸區，僅將 aggregated 統計（如月度產能、缺料事件次數）傳台供集團視角。涉及客戶 PII / 商業機密的明細資料不跨境。實務做法：陸區獨立部署模型 + Azure 大陸地區 / 鼎新雲，與台灣部署同步權重但不共享原始資料。
Q9：要不要趁機把 EAP 也整合進來重做？
A：不建議。EAP 為另案 SaaS 開發中的多租戶平台；本案網站只做：介紹 / 導流 / 試用申請 / Demo 申請 / API 串接，避免重做。EAP 上線後客戶從本網站 SSO 進入。
Q10：員工會不會擔心 AI 取代工作？
A：本案定位為「AI 輔助、不自動決策」 — 採購、廠長、IE 工程師仍是主決策者，AI 提供建議與解釋。重複機械工作（資料整理、產生報表、回覆 FAQ）由 AI 處理；員工回到判斷、設計、客戶關係等高價值工作。瀚荃學苑 Phase 5 將提供員工 AI 協作訓練。
